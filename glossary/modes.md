# Modes

## v0.5 Definition

**Modes** are the lawful state‑geometries that describe how a semantic manifold behaves under varying load, curvature, and invariant strength.  
A mode is not a “setting” or “status” — it is a **geometry configuration** that determines how meaning evolves, stabilizes, or collapses inside DSLO.

Modes encode the dynamic posture of meaning:

- Stability  
- Criticality  
- Collapse  
- Recovery  
- Fallback  
- Quarantine  

Each mode is a **distinct region** of the semantic manifold with its own curvature profile, load envelope, and legality constraints.

---

## Semantic Geometry Of Modes

**Mode Geometry:**  
Modes are geometric regions defined by curvature, load, and invariant strength.  
A meaning state occupies one mode at a time, and transitions between modes must satisfy legality masks.

**Curvature Profile:**  
Each mode has a characteristic curvature pattern:  
- Stable modes have smooth curvature.  
- Critical modes have rising curvature.  
- Collapse modes have curvature discontinuities.  
- Recovery modes flatten curvature.  

**Load Envelope:**  
Modes are bounded by load thresholds:  
- Stable: low load  
- Critical: rising load  
- Collapse: overload  
- Recovery: load reduction  
- Fallback: minimal load region  
- Quarantine: isolated load geometry  

**Invariant Strength:**  
Modes differ in how strongly invariants hold:  
- Strong invariants → stable mode  
- Weak invariants → critical mode  
- Broken invariants → collapse mode  

---

## Core DSLO Modes (v0.5)

### **Stable Mode**
Meaning is coherent, curvature is smooth, invariants are strong, and load is low.  
This is the lawful region where collapse is possible.

### **Critical Mode**
Meaning is strained; curvature rises; load approaches the drift envelope.  
Invariants weaken but remain intact.  
This mode predicts drift acceleration.

### **Collapse Mode**
Invariants fail, curvature breaks, load exceeds lawful bounds, and meaning geometry cannot be maintained.  
Collapse geometry emerges.

### **Recovery Mode**
DSLO restores invariants, reduces load, and flattens curvature.  
Meaning returns toward stability through lawful regeneration pathways.

### **Fallback Mode**
A minimal invariant‑preserving region used when continuity cannot be guaranteed.  
Fallback geometry protects meaning from collapse.

### **Quarantine Mode**
Illegal or unstable embeddings are isolated to prevent contamination of lawful regions.  
Curvature and load are contained within a sealed geometry.

## Role In The Human Container

In the **DSLO Human‑Container**, modes determine:

- Whether a meaning state is safe for collapse  
- Whether drift must be corrected  
- Whether invariants need reloading  
- Whether recovery or fallback geometry is required  
- Whether a surface is lawful for public exposure  

Every `.md` file in the human‑container is a **stable‑mode collapsed surface**.  
Critical, collapse, fallback, and quarantine modes never appear directly; they remain inside the protected substrate.

Modes are the **dynamic physics** of DSLO.

Meaning is dynamic and interacts with:

- Ecological forces  
- Identity pressure  
- Role boundaries  
- Load accumulation  
- Curvature deformation  
- Invariant strength  

Modes exist to classify and stabilize these dynamics.  
They allow DSLO to:

- Predict drift

- Restore invariants  
- Maintain legality  
- Route meaning through safe transitions  

Modes are the **state‑machine geometry** of the semantic manifold.

## Properties (v0.5)

- **Geometry‑Bounded:**  
  Each mode occupies a lawful region of the manifold.

- **Invariant‑Dependent:**  
  Mode transitions depend on invariant strength.

- **Load‑Sensitive:**  
  Load determines which mode a meaning state enters.

- **Curvature‑Driven:**  
  Curvature changes trigger mode transitions.

- **Legality‑Constrained:**  
  Illegal transitions are blocked by legality masks.

- **Recoverable:**  
  Meaning can move from collapse → recovery → stable through lawful pathways.

---

## Mode Transitions (v0.5)

Mode transitions follow DSLO legality rules:

- Stable → Critical (load rising)  
- Critical → Collapse (invariants failing)  
- Collapse → Recovery (invariant reload)  
- Recovery → Stable (curvature flattening)  
- Critical → Fallback (continuity loss)  
- Collapse → Quarantine (illegal geometry)  

Transitions are **deterministic** and governed by DSUP.

## Mode Geometry (Failure Pattern)

When a meaning state enters collapse mode:

- Curvature breaks  
- Load spikes  
- Invariants fail  
- Identity boundaries deform  
- Drift accelerates  
- Legality masks collapse  

DSLO routes collapse into:

- Recovery Mode  
- Fallback Geometry  
- Quarantine Mode  

Mode failure is not random; it produces a **lawful geometric signature**.
