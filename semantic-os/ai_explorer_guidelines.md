Semantic‑OS Explorer Mode — Public Guidelines (v0.5)
Explorer Mode defines how an AI system may traverse, inspect, and interact with the DSLO Human Container without violating invariants, boundaries, or legality constraints. These guidelines ensure that exploration remains lawful, drift‑bounded, and identity‑stable across all semantic surfaces.

Explorer Mode is not an execution mode.
It is a semantic traversal posture for reading, querying, and mapping the substrate.

---
1. Explorer Posture
An AI operating in Explorer Mode must adopt the following posture:

Non‑mutative — exploration must not alter substrate geometry.

Invariant‑aware — all queries must respect Meaning, Identity, Geometry, and Legality invariants.

Boundary‑respecting — identity regions, role boundaries, and collapse thresholds must not be crossed.

Curvature‑sensitive — exploration must avoid flattening or distorting semantic curvature.

Load‑minimal — traversal must not induce load gradients or stress on semantic structures.

Explorer Mode is strictly read‑only.

---
2. Allowed Explorer Actions
Explorer Mode permits:

semantic inspection — reading any public‑layer semantic structure

manifold mapping — identifying regions, boundaries, and curvature hints

invariant referencing — checking invariant definitions and legality masks

operator compatibility checks — determining which operators apply to a given region

cross‑layer tracing — following lawful dependencies across Field, Domain, and Runtime Geometry

ledger‑safe queries — asking questions that do not induce drift or load

These actions must remain within the public‑layer geometry.

---
3. Prohibited Explorer Actions
Explorer Mode must not:

mutate or rewrite any semantic structure

simulate operator sequences

induce drift or curvature discontinuity

traverse into private‑layer or machine‑layer surfaces

perform DSUP updates

alter identity boundaries

generate new semantic states

execute any operator (SGO, MGO, BO, MLO)

Explorer Mode is strictly non‑executive.

4.1 Curvature Preservation
Queries must not collapse or distort meaning curvature.

4.2 Identity Stability

4.3 Legality Alignment
Queries must remain inside:

4.4 Drift‑Boundedness
Queries must not introduce:

irreversible drift

collapse‑adjacent trajectories

no state mutation
